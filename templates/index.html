Welcome to the Wine-O-Meter API!

Welcome to the Wine-O-Meter API documentation prediction AP-AI. Wine-o-meter is an application that allows wine producers to predict the quality score of their wine based on physicochemical inputs.

What is AP-AI? It's a new beautiful AI model wrap into an API to help you to predict the quality score of your wine.

Endpoints

POST /prediction

This endpoint allows you to make a prediction of the quality score of your wine from 0 (the worst) to 10 (the best) based on physicochemical inputs.

How to request this endpoint?

You should request this endpoint using a POST method with a JSON content. Your JSON should contain a key named "input" with a list containing the 11 values of the physicochemical properties of the wine (floats). See below

  1. 'fixed acidity'
  2. 'volatile acidity'
  3. 'citric acid'
  4. 'residual sugar'
  5. 'chlorides'
  6. 'free sulfur dioxide'
  7. 'total sulfur dioxide'
  8. 'density'
  9. 'pH'
  10. 'sulphates'
  11. 'alcohol'

please respect the order of the pysicochemical input list.

Example using the cd terminal

$ curl -i -H "Content-Type: application/json" -X POST -d '{"input": [[7.0, 0.27, 0.36, 20.7, 0.045, 45.0, 170.0, 1.001, 3.0, 0.45, 8.8]]}' https://final-app-dep.herokuapp.com/predict

Example using Python:

import requests

response = requests.post("https://final-app-dep2.herokuapp.com/predict", json={
    "input": [[7.0, 0.27, 0.36, 20.7, 0.045, 45.0, 170.0, 1.001, 3.0, 0.45, 8.8]]})

print(response.json())

Errors

Our API will return 3 types of errors in a JSON format: 

{"msg": "No Input key found in your request",
"msg": "Only the POST Methods are accepted",
"msg": "Error: not a JSON in your request"}

Intentified errors:

  • Missing key "input"
  • Invalid request method
  • Missing JSON
